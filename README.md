# Compare and Buy Insurance Prototype

This is a mobile-first HTML prototype . The page simulates an embedded car insurance results experience inside a car buying app, with quick filters, plan cards, plan details, and a compare flow.

## What’s included

* Quick filters for common coverage needs
* Horizontally scrollable insurance plan cards
* Plan detail view with coverage, exclusions, and add-ons
* Compare mode for selecting multiple plans
* Side-by-side comparison view
* Select Plan confirmation state
* Empty state when no plans match the selected filters

## Overview

This prototype was created as part of a Product Owner assignment focused on improving the insurance comparison experience within a car buying journey.

The core problem identified in the scenario was that customers were choosing plans primarily based on price because they could not easily understand coverage differences, compare plans, or access detailed plan information.

## My Approach

Rather than redesigning the entire insurance journey, I focused on the Results Page where customers make their purchase decision.

The solution is built around three key principles:

* **Coverage Clarity** – Help customers understand what each plan includes.
* **Easy Comparison** – Allow customers to compare plans side-by-side.
* **Decision Confidence** – Guide customers toward the most suitable plan through recommendations and quick filters.

## Key Features

* Quick filters for common coverage preferences (OD, Zero Dep, Engine Cover, EV Cover, etc.)
* Coverage-focused plan cards with key highlights
* Detailed plan information accessible through an info icon
* Side-by-side comparison experience
* Recommended plan badge to help guide decision making
* Empty state handling when no plans match selected filters
